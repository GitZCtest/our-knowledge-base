## 🤝 Git 协作规范 (Git Flow)

为了避免笔记冲突和丢失，请遵守以下简单的协作流程：

### 1. 开始工作前
务必先拉取最新改动：
```bash
git pull
```

### 2. 编辑注意事项
*   **尽量在“40_个人空间”内撰写新草稿**，完成后再移动到公共区域。
*   如果需要修改公共区域的文件（如 `10_课程学习` 下的笔记），最好先在群里知会一声，或者确认对方此刻没有在修改同一文件。
*   附件（图片等）请统一放入 `99_附件` 文件夹，保持根目录整洁。

### 3. 提交更改
完成一个阶段的工作后，及时提交：

```bash
git add .
git commit -m "你的更新说明，例如：新增了数据结构笔记 / 修改了项目计划"
git push
```

### 4. 遇到冲突 (Conflict)
如果 `git pull` 提示冲突：
1.  打开冲突文件，Obsidian 会显示 `<<<<<<< HEAD` 等标记。
2.  保留需要的内容，删除 Git 标记符号。
3.  重新 `git add` -> `commit` -> `push`。
